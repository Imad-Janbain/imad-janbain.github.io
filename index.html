§ 01

About.

I became a machine learning researcher by accident. I trained as a mechatronics engineer, wrote a master’s thesis on Gaussian Processes for rural smart grids, and then spent a PhD turning the Seine River into a deep learning problem.

Most of my work sits at the intersection of two deeply unfashionable things: time series and messy real-world data. It’s the kind of ML that doesn’t get written up in Twitter threads — but it’s the kind that environmental agencies actually deploy. I’ve built production-grade reconstruction pipelines that recovered decades of missing measurements, flood-risk forecasters for DREAL, and a methodology (Mini-LookBack) that cut training time by 96% and is now on its second generation of derived research.

I’m in the final stretch of a postdoc applying Kolmogorov–Arnold Networks to long-term groundwater forecasting under CMIP6 climate scenarios — the first systematic KAN-vs-LSTM evaluation on hydroclimatic data. In Autumn 2026 I’m moving to Paris for an industry role where rigorous ML meets real-world signal.

§ 02

Selected work.

A few things I’ve built that I’m proud to put my name on.

  1. 01

    seine-water-level-imputation

    2023 → 2026

    A production-grade Python package for reconstructing missing river water-level data with LSTM / GRU and the Mini-LookBack decomposition from my 2023 Hydrological Sciences Journal paper. Ships with a synthetic tidal-harmonic toy-data generator, an Optuna tuning workflow with SQLite persistence, an auto-running data-quality gate with markdown reports, Docker, CI/CD, and an electricity-demand worked example. Opinionated defaults; deep configurability where it matters.

    • Python
    • TensorFlow
    • Optuna
    • Docker
    • CI/CD
    • MLflow
    • pytest
    View on GitHub
  2. 02

    deep-ts-imputer

    2024 → 2026

    A generalized deep-learning framework for multivariate time-series imputation, based on my 2023 Seine water-quality paper. Five architectures (LSTM, BiLSTM, GRU, BiGRU, CNN-BiLSTM with optional attention), a two-phase methodology — a grid of model experiments, then progressive reconstruction with feasibility-gated model selection — and depth-aware spatial EDA for surface-vs-bottom probes. Units and column aliases are first-class citizens.

    • Python
    • PyTorch
    • Attention
    • Matplotlib
    • Scikit-learn
    • pandas
    View on GitHub
  3. 03

    Mini-LookBack — input decomposition for LSTMs

    2023

    A novel input decomposition that exploits temporal structure before the model sees it — cutting LSTM training time by 96% and improving forecast accuracy by 83% (NSE metric vs. a standard LSTM baseline on multi-decadal Seine River data). Published in Hydrological Sciences Journal; now the methodological backbone of two follow-up manuscripts under review.

    • Python
    • TensorFlow
    • Time-series decomposition
    • NSE evaluation
    Read the paper
  4. 04

    SAR flood segmentation — Sentinel-1

    2022

    A self-initiated computer-vision project: semi-supervised semantic segmentation on Sentinel-1 SAR imagery for flood-extent mapping. Pseudo-labeling to handle label-scarce satellite data; evaluated with IoU and F1; packaged as a reusable Python CLI. Built to deepen my computer-vision & remote-sensing fluency.

    • Python
    • PyTorch
    • OpenCV
    • Sentinel-1 SAR
    • Semi-supervised learning
